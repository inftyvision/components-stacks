# MDX Viewer Component

The MDX Viewer is a powerful documentation viewer that renders MDX content with rich formatting, syntax highlighting, and page navigation.

## Features

- Rich MDX rendering
- Custom theme support
- Page-based navigation
- Code syntax highlighting
- Responsive design
- Keyboard navigation
- Custom typography

## Installation

```bash
npm install @mdx-js/react react-markdown remark-gfm
```

## Basic Usage

```typescript
import MdxViewer from '../components/MdxViewer';

function Documentation() {
  return (
    <MdxViewer content={`
      # Introduction
      
      Welcome to the documentation.
      
      ## Getting Started
      
      Here's a code example:
      
      \`\`\`typescript
      function hello(name: string) {
        console.log(\`Hello, \${name}!\`);
      }
      \`\`\`
    `} />
  );
}
```

## Content Structure

The viewer splits content into pages based on h1 headers:

```markdown
# Page 1 Title
Content for page 1...

## Section 1.1
More content...

# Page 2 Title
Content for page 2...

## Section 2.1
More content...
```

## Supported Markdown Features

1. **Headers**
   ```markdown
   # H1 Title
   ## H2 Subtitle
   ### H3 Section
   ```

2. **Emphasis**
   ```markdown
   *italic text*
   **bold text**
   ***bold italic text***
   ```

3. **Lists**
   ```markdown
   - Unordered item
     - Nested item
       - Deep nested item
   
   1. Ordered item
      1. Nested ordered
      2. Another nested
   ```

4. **Code Blocks**
   ````markdown
   ```typescript
   function example() {
     return "Hello, World!";
   }
   ```
   ````

5. **Blockquotes**
   ```markdown
   > Single quote
   > > Nested quote
   ```

6. **Tables**
   ```markdown
   | Header 1 | Header 2 |
   |----------|----------|
   | Cell 1   | Cell 2   |
   ```

## Keyboard Navigation

- `←` / `→`: Navigate between pages
- `Home`: Go to first page
- `End`: Go to last page
- `Ctrl + ←`: Previous page
- `Ctrl + →`: Next page

## Best Practices

1. **Content Organization**
   - Use h1 headers for main pages
   - Keep content focused
   - Use descriptive titles
   - Include examples

2. **Code Examples**
   - Use syntax highlighting
   - Keep examples concise
   - Include comments
   - Show usage patterns

3. **Styling**
   - Use consistent formatting
   - Follow spacing guidelines
   - Use theme variables
   - Test responsiveness

## Troubleshooting

   - Verify theme variables
   - Check CSS specificity
   - Inspect element styles

3. **Navigation Problems**
   - Check h1 headers
   - Verify content structure
   - Check event handlers
